# status/ ディレクトリガイド

実装ステータスと将来ロードマップを管理する。

---

## 1. 目的

設計書（`docs/design/`）から実装ステータスを分離し、一元管理する。設計書はコードの**あるべき姿**を記述し、本ディレクトリは**現在の進捗と将来計画**を記述する。

## 2. ファイル構成

| ファイル | 内容 |
|---------|------|
| `implementation.md` | モジュールごとの機能実装状態 |
| `roadmap.md` | 将来の拡張計画（未着手の機能） |

## 3. 記法規約

### ステータス

| ステータス | 意味 |
|-----------|------|
| `done` | 実装・テスト完了 |
| `partial` | 一部実装済み（詳細を備考に記載） |
| `not-started` | 未着手 |

### 更新タイミング

- 機能の実装が完了したとき → `implementation.md` を更新
- 新たな将来計画が決定したとき → `roadmap.md` を更新
