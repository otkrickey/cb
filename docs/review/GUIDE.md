# レビューガイドライン作成ガイド

`docs/review/modules/` 配下のモジュール別レビューガイドラインの記述規約。

## ディレクトリ構成

```
docs/review/
├── GUIDE.md          # 本ファイル（記述規約）
├── TEMPLATE.md       # モジュールガイドラインのテンプレート
├── FORMAT.md         # 共通出力形式・分類基準
└── modules/
    ├── cb-core.md    # Rustコアクレート
    ├── ui.md         # Swift/SwiftUI UI層
    └── docs.md       # ドキュメント整合性
```

## 役割分担

| ファイル | 内容 | 変更頻度 |
|---------|------|---------|
| `FORMAT.md` | 出力形式・分類基準（全モジュール共通） | 低（形式変更時のみ） |
| `modules/{module}.md` | モジュール固有の評価項目 | 中（機能追加・設計変更時） |
| `TEMPLATE.md` | 新規モジュール追加時のひな形 | 低 |

## 記述ルール

### モジュールファイル (`modules/{module}.md`)

1. **見出し**: `# {Module名} レビューガイドライン`
2. **対象セクション**: レビュー対象のファイルパスを明記する
3. **評価項目セクション**: 番号付きリストで記載する
   - 各項目: `**項目名** — 確認内容の説明` の形式
   - 項目は具体的かつ検証可能な記述にする
4. **出力形式セクションは不要**: `FORMAT.md` に統一されているため、個別ファイルには含めない

### 評価項目の粒度

- 1項目 = 1つの検証観点
- 抽象的な項目（「コード品質」等）は避け、具体的な検証対象を明記する
- 設計書への参照がある場合はパスを含める

### モジュール追加時

1. `TEMPLATE.md` をコピーして `modules/{module}.md` を作成
2. 対象パスと評価項目を記入
