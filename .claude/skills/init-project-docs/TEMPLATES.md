# ドキュメントテンプレート集

init-project-docs スキルが生成する各ファイルのテンプレート。プロジェクト固有の情報で `{placeholder}` を置換して使用する。

---

## design/GUIDE.md テンプレート

主要セクション構成:

```
1. 目的（対象読者、文書化の方針、実装ステータスに関する原則）
2. ディレクトリ構成
3. 設計書の種類と役割（modules / flows / decisions / usecase_mapping）
4. 種別ごとのセクション取捨選択表
5. 記法規約（Mermaid図、コード参照形式、相互参照）
6. AIエージェント向けルール（調査の深さ、作成手順）
7. テンプレートの使用方法（ファイル命名規則）
8. 品質チェックリスト
9. 参考資料
10. 更新履歴
```

**カスタマイズポイント**:
- コード参照形式をプロジェクトの言語に合わせる
- ディレクトリ構成に選択されたカテゴリのみ記載する

## plans/GUIDE.md テンプレート

主要セクション構成:

```
1. 目的（ギャップ分析、優先度付け、スコープ定義、チーム作業計画）
2. ディレクトリ構成（ライフサイクル: アクティブ → resolved/）
3. 実装計画の種別（bugfix / missing_docs / enhancement / refactoring）
4. ドキュメント構成（必須/推奨セクション表）
5. 記法規約（メタデータ、タスクID、タスクリスト、スコープ境界）
6. AIエージェント向けルール
7. 品質チェックリスト
8. テンプレートの使用
9. 参考資料
```

## review/ テンプレート群

### GUIDE.md
- ディレクトリ構成、役割分担、記述ルール、モジュール追加手順

### FORMAT.md
- 出力形式: Critical / Medium / Low / 確認済み

### TEMPLATE.md（modules/用）
- 対象パス + 評価項目（番号付きリスト）
